Part 1: Core Concepts & Installed Libraries
Before looking at the code, let us look at the foundation. I installed three core ecosystems:

1. What is child_process and spawn?

      Node.js runs on a single thread. It is designed for lightweight tasks like routing web traffic, not for heavy 
      processing. Python, however, excels at data parsing and AI computing.
      * child_process is a built-in Node.js module 
        that lets your Javascript engine open a separate terminal process inside your operating system.
      * spawn launches a background terminal window running a specific command (like launching your Anaconda Python environment) and
        opens a live data communication line (a stream) between Node.js and Python.

2. youtube-transcript-api

    This Python library is designed to bypass standard webpage scraping entirely. Instead of opening a web browser to download the YouTube page's HTML structure, it uses backend tokens to securely request the text subtitle data stream directly from YouTube’s hidden XML/JSON database endpoints.


3. @google/genai vs google-genai

     * Node.js Side (@google/genai): A Javascript wrapper used to bundle parameters, connect securely over HTTPS, and stream queries directly into Google’s Gemini 2.5 architecture.

     * Python Side (google-genai): The identical framework mapped directly to Python syntax rules.

function extractVideoId(url) {
  const regExp = /^.*(?:(?:youtu\.be\/|v\/|vi\/|u\/\w\/|embed\/|shorts\/)|(?:(?:watch)?\?v=|\&v=))([^#\&\?]*)/;
  const match = url.match(regExp);
  return (match && match[1] && match[1].length === 11) ? match[1] : null;
}
